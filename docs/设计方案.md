## 2. 顶层架构

```
浏览器 (Vue 3 SPA, Element Plus)
   │  fetch + credentials: 'include'，X-CSRFToken 头
   ▼
Flask 应用 (单体, 应用工厂模式)
   ├─ /api/auth/*        登录、登出、当前用户、CSRF token
   ├─ /api/members/*     会员 CRUD（admin/staff）
   ├─ /api/cards/*       办卡、续费、冻结、查询
   ├─ /api/coaches/*     教练 CRUD（admin/staff）
   ├─ /api/classes/*     课程定义 + 排课
   ├─ /api/bookings/*    预约、取消（会员自助 + admin）
   ├─ /api/attendance/*  签到、出勤查询
   ├─ /api/stats/*       统计汇总（会员数、收入、出勤率）
   ├─ /api/exports/*     CSV / 打印视图
   └─ Flask-Login session  +  Flask-WTF CSRF
   ▼
SQLAlchemy ORM（对照 Spring Data JPA）
   ▼
SQLite (instance/webgym.db，Alembic 管理 schema)
```

**架构分层（论文里画图并与 Spring 三层对照）**：

| 本项目（Flask） | Spring Boot 对应 | 职责 |
|---|---|---|
| `app/api/*.py` 蓝图 | `@RestController` | HTTP 入口、参数校验、序列化 |
| `app/services/*.py` | `@Service` | 业务规则、事务边界、状态机 |
| `app/models/*.py` (SQLAlchemy) | `@Entity` + `Repository` | 持久化、关系映射 |
| `app/utils/auth.py` 装饰器 | Spring Security `SecurityFilterChain` | 鉴权、角色校验 |
| `app/extensions.py` | Spring `@Configuration` Bean | 扩展实例化 |

## 4. 数据模型

```
User
  id, username (uniq), password_hash, role: Enum[admin, staff, member],
  is_active, created_at, last_login_at

MemberProfile  (member 角色独有)
  user_id (PK/FK→User), real_name, gender, phone (uniq, idx),
  birthday, id_card (uniq, Fernet 加密存储), emergency_contact, note

Coach
  id, name, gender, phone, specialty (text, e.g. "瑜伽,普拉提"),
  bio, hired_at, is_active

CardType        # 卡类型字典
  id, name, duration_days (nullable), total_visits (nullable),
  price, is_active

MembershipCard  # 会员持有的卡实例
  id, card_no (uniq, 业务编号), member_id→User,
  card_type_id→CardType, start_date, end_date,
  remaining_visits (nullable, 次卡用),
  status: Enum[active, frozen, expired, cancelled],
  issued_by→User(staff), issued_at, frozen_at (nullable)

ClassDefinition # 课程定义
  id, name, description, coach_id→Coach,
  capacity (默认), duration_minutes, is_active

ClassSession    # 一次具体排课
  id, class_def_id→ClassDefinition, coach_id→Coach,
  start_at, end_at, capacity (覆盖默认), location,
  status: Enum[scheduled, cancelled, finished]

Booking
  id, member_id→User, session_id→ClassSession,
  card_id→MembershipCard,
  status: Enum[booked, cancelled, attended, no_show],
  booked_at, cancelled_at, source: Enum[self, admin]
  UNIQUE (member_id, session_id) WHERE status != cancelled

Attendance
  id, booking_id→Booking (uniq), checked_in_at,
  checked_in_by→User
```

**业务约束（service 层强制）**：

- 预约：会员当前必须至少一张 `status=active` 且未过期的卡；次卡 `remaining_visits>0`；目标 session 未取消且未满员；同会员同 session 不能重复有效预约。
- 取消：开课前 2 小时内会员不能自助取消（admin 可绕过）。
- 签到：只能为本人的有效预约签到；session `status=finished` 后不再允许签到。
- 状态机：`MembershipCard.status` 由 service 切换（active ⇄ frozen，active → expired/cancelled），不允许从路由层直接 PATCH。

## 5. 权限模型

| 角色 | 能做什么 |
|---|---|
| `admin` | 全部 |
| `staff` | 除"用户/角色管理"外都能 |
| `member` | 看自己的卡 + 预约；自助预约/取消；查自己签到记录；改自己资料 |

实现：

- `User.role` 枚举。
- `app/utils/auth.py` 的 `@role_required(*roles)` 装饰器（论文里与 Spring Security 的 `@PreAuthorize("hasRole(...)")` 对照）。
- 按 id 查会员资源的接口要二次校验 `current_user.role in (admin,staff) or resource.member_id == current_user.id`，防 IDOR。
- 前端 `router.meta.roles` 拦截只是 UX；后端是真正闸门。

## 6. 安全设计（对应任务书 §二.3）

| 防护点 | 实现 | Spring 对照（论文用） |
|---|---|---|
| 身份认证 | Flask-Login session | Spring Security `UsernamePasswordAuthenticationFilter` |
| 授权 | `@role_required` 装饰器 + 资源所有权校验 | Spring `@PreAuthorize`、`AccessDecisionManager` |
| CSRF | Flask-WTF CSRFProtect，`GET /api/auth/csrf-token` 下发 | Spring `CsrfFilter`（默认开） |
| 密码 | `werkzeug.security` pbkdf2:sha256 | Spring `BCryptPasswordEncoder` |
| 敏感字段 | 身份证号 Fernet 加密入库 | Spring `@Convert` + AES |
| Session cookie | `HttpOnly`, `SameSite=Lax`, 生产 `Secure` | 同 Spring 默认 |
| 速率限制 | Flask-Limiter，`/api/auth/login` 5/min | Spring + Bucket4j |
| CORS | 仅允许前端 dev/prod origin | Spring `CorsConfigurationSource` |
